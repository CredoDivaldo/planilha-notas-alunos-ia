## Story Statement

**As a** professor,  
**I want** a grade management table with inline cell editing, CSV import, and automatic final-grade calculation,  
**so that** I can enter, correct, and verify each student's grades per evaluation component before publishing results.

---

## Acceptance Criteria

- [ ] **AC1:** A página `/notas` renderiza com breadcrumb "Notas > [Turma · Disciplina · Semestre · Turno]" e link "◀ Mudar contexto"; o contexto activo é herdado da `ContextBar` ou seleccionado via Select no breadcrumb
- [ ] **AC2:** A toolbar tem botões: "📥 Importar CSV" (abre modal), "💾 Guardar alterações" (desactivado quando não há modificações pendentes), "🧮 Recalcular" (recalcula Nota Final), "📤 Publicar notas" (desactivado se há notas incompletas) + badge de contagem de notas por preencher
- [ ] **AC3:** A tabela mostra colunas dinâmicas baseadas nos componentes do contexto: Nº, Nome, [Componente1 (Peso%)], [Componente2 (Peso%)], …, Nota Final, Estado; as colunas de componentes são geradas dinamicamente a partir de `GET /academic-contexts/{id}`
- [ ] **AC4:** Duplo-clique numa célula de componente abre edição inline: a célula substitui o valor por um `<input type="number" min="0" max="20">` com botões ✅ Guardar e ✕ Cancelar ao lado da linha
- [ ] **AC5:** Ao confirmar edição (✅ ou Enter), chama `PATCH /grades/{id}` com o novo valor; durante o pedido, a linha entra em estado `saving` (spinner, desactivada); em caso de erro, a linha entra em estado `error` com mensagem inline
- [ ] **AC6:** A Nota Final é calculada automaticamente no cliente após cada edição confirmada: `notaFinal = sum(componente_i * peso_i / 100)`, arredondada a 1 casa decimal; só é exibida quando todos os componentes estão preenchidos
- [ ] **AC7:** Os estados de linha são visualmente distintos: `default` (fundo branco), `editing` (fundo azul claro `#EFF6FF`), `saving` (fundo cinza, spinner), `error` (fundo vermelho claro `#FEF2F2`, borda `#B91C1C`)
- [ ] **AC8:** Os badges de Estado são: "Lançada" (verde), "Incompleta" (âmbar — falta algum componente), "Vazio" (vermelho — nenhum componente preenchido), "Publicada" (azul — já publicada, imutável)
- [ ] **AC9:** Linhas com estado "Publicada" não podem ser editadas; duplo-clique não activa inline edit
- [ ] **AC10:** A barra de filtros tem: campo de pesquisa por nome/número (tempo real, cliente), Select "Mostrar" com opções: Todos, Completos, Incompletos, Vazios; Select "Ordenar" com opções: Número, Nome, Nota Final
- [ ] **AC11:** O modal "📥 Importar CSV" tem: Select para escolher o componente de avaliação alvo, `FileDropzone` para arrastar/soltar o CSV, pré-visualização das primeiras 3 linhas, resumo de validação (X registos válidos + Y sem correspondência); botão "Importar X notas" chama `POST /grades/upload`; abre via `POST /grades/upload`
- [ ] **AC12:** As `StatCards` em baixo da página mostram: Nota média, Nota mais alta, Nota mais baixa, Aprovados (N e %), Reprovados (N e %), Incompletos (N e %)
- [ ] **AC13:** O histórico de importações mostra as últimas 3 importações com timestamp e contagem de notas carregadas; botão "Ver log completo" expande
- [ ] **AC14:** O botão "📤 Publicar notas" redireciona para `/publicar` (Story 7.7) com o contexto activo pré-seleccionado; está desactivado com tooltip se houver notas incompletas
- [ ] **AC15:** A tabela é acessível WCAG AA: `<th scope="col">` em cabeçalhos, `scope="row"` no Nº do estudante; células em edição têm `aria-label="Nota de Frequência para Ana Silva"`

### Estados de Linha GradeTable (crítico — do component-inventory.md)

```ts
type RowStatus = 'default' | 'editing' | 'saving' | 'error'
```

- `default`: valores estáticos, cursor pointer no hover
- `editing`: célula activa com `<input>` + botões save/cancel; fundo `#EFF6FF`
- `saving`: spinner, toda a linha disabled; fundo cinza
- `error`: borda `#B91C1C`, mensagem de erro inline; re-tentar disponível

### Endpoints API Relevantes

| Endpoint | Método | Uso |
|----------|--------|-----|
| `GET /academic-contexts/{id}` | GET | Obter componentes de avaliação do contexto |
| `GET /grades/` | GET | Listar notas com param de contexto |
| `PATCH /grades/{id}` | PATCH | Actualizar nota individual |
| `POST /grades/upload` | POST | Importar CSV de notas |
| `POST /grades/match` | POST | Regenerar match se necessário |

### Cálculo de Nota Final

```ts
function calcNotaFinal(components: {value: number, weight: number}[]): number | null {
  if (components.some(c => c.value === null)) return null
  return components.reduce((sum, c) => sum + (c.value * c.weight / 100), 0)
}
```
