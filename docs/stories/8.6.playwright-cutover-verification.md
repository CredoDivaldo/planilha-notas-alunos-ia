## Notes for future work

- The Playwright spec is intentionally resilient to a missing backend. When the dev stack is up, all 9 tests run end-to-end. When it's not, the happy-path tests skip gracefully and the 3 regression tests still run (because they hit FastAPI directly on port 8000 — which is reachable independently of the dev script).
- A full Playwright run with the dev stack takes ~30s. That's outside the scope of this story's CI gate (which uses `--list` only), but easy to wire up in `.github/workflows/e2e.yml` if the team wants a true end-to-end CI gate.
- The `FALLBACK_*` rename is more than cosmetic — the original `MOCK_` prefix leaked a "test fixture" mental model that didn't match the data's actual role. FALLBACK_* accurately describes "data used when the API is offline in dev" and aligns with the same convention used in `AuthContext.tsx` (the comment there is `// API unavailable — mock login for development`).
