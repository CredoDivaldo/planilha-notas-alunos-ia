## Story Statement

**As a** professor,  
**I want** an operational dashboard with 5 sequential step cards (Upload Students → Upload Grades → Match → Connect WhatsApp → Send),  
**so that** I can complete the grade-sending workflow in a guided, gated sequence where each step only unlocks after the previous one is completed.

---

## Acceptance Criteria

- [ ] **AC1:** A página `/painel` renderiza com o layout DashboardLayout: `AppHeader` (nav completo), `ContextBar` (selector de contexto activo + botão "Novo"), área principal com 2 colunas (70% + 30%)
- [ ] **AC2:** O `AppHeader` tem navegação horizontal: "Painel", "Contextos", "Notas", "Calendário", "Publicar" + avatar e nome do professor + botão "Sair"
- [ ] **AC3:** A `ContextBar` mostra um Select com os contextos académicos do professor (carregados via `GET /academic-contexts/`); a selecção persiste durante a sessão e filtra toda a página
- [ ] **AC4:** O Progress Stepper mostra os 5 passos numerados com estado visual correcto: `completed` (ícone ✅ verde), `active` (borda azul pulsante), `locked` (ícone 🔒 cinza), `error` (ícone ❌ vermelho)
- [ ] **AC5:** O Step Card activo renderiza com borda `#0D6EFD`, corpo expandido, lista de pré-condições satisfeitas e botão de acção disponível
- [ ] **AC6:** Os Step Cards bloqueados (`locked`) estão colapsados e listam as pré-condições em falta; não são interactivos
- [ ] **AC7:** Step 1 — Upload Estudantes: `FileDropzone` aceita ficheiro CSV; submete via `POST /students/upload`; exibe contagem de estudantes carregados; ao concluir, avança para Step 2
- [ ] **AC8:** Step 2 — Upload Notas: `FileDropzone` aceita ficheiro CSV; submete via `POST /grades/upload`; exibe contagem de notas carregadas; ao concluir, avança para Step 3
- [ ] **AC9:** Step 3 — Gerar Match: botão "🔄 Gerar Match" chama `POST /grades/match`; exibe tabela de resultados (max 5 linhas + "Ver todos" que abre modal); mostra timestamp do último match; ao concluir sem erros, avança para Step 4
- [ ] **AC10:** A tabela de match mostra: Nº Estudante, Nome, Nota, Status (✅ Match / ❌ Sem nota / ⚠️ Sem telefone)
- [ ] **AC11:** Step 4 — Conectar WhatsApp: sub-passos Criar/Conectar/Verificar; mostra QR Code com countdown de 45s; polling a cada 5s para verificar estado (reutiliza padrão da Story 4.3); ao conectar, avança para Step 5
- [ ] **AC12:** O QR Code exibe um countdown visual; botão "🔄 Gerar novo QR" regenera quando expirado
- [ ] **AC13:** Step 5 — Disparar Mensagens: mostra template de mensagem editável; selector de modo (Simulação / Envio Real); botão "🚀 Disparar"; modo Envio Real abre modal de confirmação antes de enviar via `POST /broadcast/`
- [ ] **AC14:** O modal de confirmação de Envio Real mostra: número de destinatários, aviso de irreversibilidade, checkbox obrigatória, botão "Confirmar Envio Real"
- [ ] **AC15:** A sidebar direita mostra `QuickStatsSidebar` com StatCards actualizados após cada operação: Estudantes, Notas, Matched, Sem match, Tel. inválido, Enviados, Falhas
- [ ] **AC16:** A sidebar direita mostra o painel `WhatsApp Status` com badge verde/vermelho, nome da instância e botão "Verificar estado" (polling a cada 30s)
- [ ] **AC17:** State gating: step N+1 permanece `locked` enquanto step N não estiver `completed`; o estado persiste em `sessionStorage` para sobreviver a refreshes durante a sessão

### Estados do StepCard (crítico)

```ts
type StepStatus = 'locked' | 'active' | 'completed' | 'error'
```

- `locked`: header cinza, corpo colapsado, lista de pré-condições pendentes
- `active`: borda `#0D6EFD` 2px, header azul, corpo expandido, acções disponíveis
- `completed`: borda `#15803D`, header verde, corpo colapsado, resumo visível, clicável para expandir
- `error`: borda `#B91C1C`, mensagem de erro inline, botão "Re-tentar"

### Endpoints API Relevantes

| Endpoint | Método | Step |
|----------|--------|------|
| `GET /academic-contexts/` | GET | ContextBar |
| `POST /students/upload` | POST | Step 1 |
| `POST /grades/upload` | POST | Step 2 |
| `POST /grades/match` | POST | Step 3 |
| `POST /broadcast/` | POST | Step 5 |

### Reutilização da Story 4.3

A Story 4.3 (`docs/stories/4.3.qr-polling.md`) já implementou a lógica de QR polling. O Step 4 deve reutilizar o padrão estabelecido. Verificar se existe código reutilizável em `public/` ou `src/` antes de reimplementar.

### Acessibilidade

- `StepCard` com `role="region"` + `aria-labelledby` apontando para o título do step
- `GradeTable` dentro do Step 3: `<th scope="col">` em todas as colunas
- `aria-live="polite"` nos StatCards da sidebar para actualizações de valores
- Modal de confirmação (Step 5): focus trap, Escape fecha, `role="dialog"` com `aria-labelledby`
- Progress stepper: `aria-current="step"` no step activo
