## Story

**As a** student,  
**I want** to ask questions about my grades in natural language via WhatsApp,  
**so that** I receive a clear, accurate answer based on my published grades without needing to access the portal.

---

## Acceptance Criteria

### AC-1: AI model is called with student grade context
**Given:** a student message and a valid student record with published grades  
**When:** the AI grade query service is invoked  
**Then:** it builds a system prompt containing the student's published grades and calls Baidu QianFan API

### AC-2: Response is in Portuguese and grade-specific
**Given:** the student asks "Qual é a minha nota de Matemática?"  
**When:** the AI service generates a response  
**Then:** the response is in Portuguese, references the correct published grade, and does not invent data not present in the context

### AC-3: Unpublished grades are not exposed
**Given:** a subject exists but has no current publication snapshot  
**When:** the student asks about that subject  
**Then:** the AI response informs the student that grades for that subject have not yet been published, without revealing any internal values

### AC-4: Student with no published grades receives a graceful response
**Given:** the student has no current publication snapshots at all  
**When:** the AI service is invoked  
**Then:** it returns a message informing the student that no grades have been published yet, without calling the AI API unnecessarily

### AC-5: AI API failure is handled without crashing
**Given:** the AI API is unavailable or returns an error  
**When:** the service attempts to generate a response  
**Then:** it returns a fallback message ("Não foi possível processar o teu pedido agora. Tenta mais tarde.") and logs the error

### AC-6: Dry-run endpoint returns AI response without sending WhatsApp
**Given:** the professor calls `POST /api/v1/chatbot/test`  
**When:** a valid student number and message are provided  
**Then:** the system returns the AI-generated response and the grade context used, without sending any WhatsApp message

## Scope

### IN
- Python service `backend/app/services/ai_chatbot.py`.
- Query to `publication_snapshots` / student read model for published grades.
- System prompt construction with student grade context.
- AI API call (Baidu QianFan via `ernie-bot-sdk`; configured by `BAIDU_API_KEY` env var).
- Response in Portuguese.
- Graceful handling when no published grades exist.
- Error fallback message.
- Endpoint `POST /api/v1/chatbot/test` (dry-run, professor role only).

### OUT
- Sending WhatsApp reply (Story 6.3).
- Multi-turn conversation memory.
- Grade editing, correction or administrative actions via chatbot.
- Any exposure of internal `grade_entries` or unpublished data.

## Risks

| Risk | Mitigation |
|------|------------|
| AI invents grades not in context | System prompt explicitly forbids answering outside the provided context; tested with adversarial prompts |
| AI API key missing in dev | `AI_API_KEY` absent → return mock response in development; fail clearly in production |
| Prompt injection via student message | Sanitise student message before insertion into prompt; system prompt sets strict boundaries |
| AI response too long for WhatsApp | Truncate to 1000 characters if needed; response format enforced in system prompt |

## Dev Notes

### System prompt structure
```
És um assistente académico. Responde APENAS com base nos dados de notas fornecidos abaixo.
NÃO inventes notas, estados ou datas que não estejam nos dados. Responde em Português.
Se a pergunta não for sobre notas académicas, redireciona educadamente.

Dados publicados do estudante {student_number}:
{grades_context}

Pergunta do estudante: {student_message}
```

### Grade context format
```
- Matemática | Semestre 2026-1 | Turma A | Nota: 17 | Estado: Aprovado | Publicado: 2026-05-28
- Programação | Semestre 2026-1 | Turma A | Nota: 15 | Estado: Aprovado | Publicado: 2026-05-28
```
