## Story Statement

**As a** student,  
**I want** a mobile-first portal that shows my published grades, a mini calendar of upcoming academic events, and a WhatsApp chatbot shortcut,  
**so that** I can check my results and exam schedule at any time without seeing any unpublished or other students' data.

---

## Acceptance Criteria

- [ ] **AC1:** A página `/portal` é acessível após login com role `estudante`; qualquer tentativa de acesso com outro role redireciona para o painel correcto
- [ ] **AC2:** O `AppHeader` mostra: "📊 Portal do Estudante", nome do estudante, número de estudante e botão "Sair"
- [ ] **AC3:** Um banner de boas-vindas exibe "👋 Olá, [Nome]!" com contexto académico activo: turma, disciplina, semestre, turno
- [ ] **AC4:** A secção "Minhas Notas" mostra, para cada disciplina com notas publicadas: nome da disciplina, nome do docente, semestre, tabela de componentes (Componente | Peso% | Nota | Estado), Nota Final em destaque e badge de resultado ("✅ APROVADO(A)" verde ou "❌ REPROVADO(A)" vermelho)
- [ ] **AC5:** Disciplinas com notas **não publicadas** mostram apenas: nome da disciplina e mensagem "⏳ As notas desta disciplina ainda não foram publicadas pelo docente. Receberá uma notificação via WhatsApp quando disponíveis."
- [ ] **AC6:** Nenhuma nota não publicada é visível ou inferível — a API `GET /student-portal/grades` já filtra; o frontend não deve exibir valores `null` como zero ou campo vazio que sugira existência de nota
- [ ] **AC7:** Botão "📥 PDF" no cabeçalho da secção de notas gera/descarrega um PDF com as notas publicadas do semestre actual
- [ ] **AC8:** O mini-calendário mensal mostra o mês actual com EventDots coloridos por tipo; navegação anterior/próximo mês via setas; clicar num evento exibe tooltip com: data, tipo, hora e local
- [ ] **AC9:** A lista "Próximos Eventos" mostra os eventos dos próximos 30 dias agrupados por data, carregados via `GET /student-portal/calendar`; cada evento mostra: data, tipo colorido, descrição, hora e local
- [ ] **AC10:** O banner "Chatbot WhatsApp" mostra texto "💬 Tem dúvidas sobre as suas notas?" e botão "Abrir WhatsApp" que abre `https://wa.me/{numero_chatbot}` em nova tab; o número é configurável via variável de ambiente `VITE_WHATSAPP_CHATBOT_NUMBER`
- [ ] **AC11:** O layout é mobile-first: em viewport ≤ 640px, as colunas de notas e calendário empilham-se verticalmente; a tabela de notas tem scroll horizontal; todos os botões são touch-friendly (min. 44x44px)
- [ ] **AC12:** Em viewport 320px, a página é utilizável sem overflow horizontal ou elementos cortados
- [ ] **AC13:** Se a sessão expirar, o utilizador é redirecionado para `/login?expired=true` com mensagem "A sua sessão expirou. Por favor, entre novamente."

## Dev Notes

### Wireframe de Referência
**Ficheiro:** `outputs/wireframes/planilha-notas/screens/03-portal-estudante.md`

O wireframe define:
- Header com nome + número do estudante + logout
- Banner de boas-vindas com contexto académico
- Layout 2 colunas (65% notas + 35% calendário + eventos) em desktop
- Tabela de componentes: Componente | Peso | Nota | Estado
- Resultado em destaque: "✅ APROVADO(A)" ou "❌ REPROVADO(A)"
- Variante nota não publicada: mensagem amigável sem revelar valor
- Mini-calendário mensal com EventDots
- Lista de próximos eventos
- Banner WhatsApp chatbot no fundo

### Endpoints API Relevantes

| Endpoint | Método | Uso |
|----------|--------|-----|
| `GET /student-portal/grades` | GET | Notas publicadas do estudante autenticado |
| `GET /student-portal/calendar` | GET | Eventos do calendário do contexto do estudante |

### Segurança de Dados (crítico)

- O frontend usa **apenas** os dados retornados por `GET /student-portal/grades`
- A API do Epic 5 (Story 5.5) já filtra para retornar só `estado = publicado`
- O frontend **não deve** fazer chamadas a `GET /grades/` (endpoint do professor)
- Se um campo de nota vier `null` da API, mostrar "—" (traço), nunca "0" ou campo vazio

### Layout Responsivo

```
Desktop (>= 1280px): grid grid-cols-[65%_35%]
Tablet (640-1279px): grid grid-cols-[60%_40%]
Mobile (< 640px):    flex flex-col (notas em cima, calendário em baixo)
```

### Acessibilidade

- Tabela de componentes: `<th scope="col">` + `<th scope="row">` para nome do componente
- EventDots: `<button aria-label="17 Jun - Exame Final Inglês">` com tooltip via `title` ou `role="tooltip"`
- Banner chatbot: botão "Abrir WhatsApp" com `target="_blank" rel="noopener noreferrer"` + `aria-label="Abrir chatbot no WhatsApp (abre nova janela)"`
- `aria-live="polite"` no container de notas para actualizações após login
