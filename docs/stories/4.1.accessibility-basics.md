## Story
**As a** utilizador com necessidades de acessibilidade,
**I want** a aplicação a seguir padrões WCAG 2.2 AA básicos,
**so that** possa utilizar a interface com tecnologias assistivas e navegação por teclado.

## Acceptance Criteria
1. Todos os elementos interactivos têm `aria-label` ou `aria-labelledby` adequados
2. Indicadores de foco visíveis em todos os elementos focáveis (outline não removido)
3. Estrutura HTML semântica: uso correcto de `<header>`, `<main>`, `<section>`, `<nav>`, `<button>`, `<label>`
4. Todos os campos de formulário têm `<label>` associado via `for`/`id`
5. Imagens decorativas têm `alt=""`, imagens informativas têm `alt` descritivo
6. Ordem de tab lógica e sequencial (não usa `tabindex` > 0)
7. Contraste de cores mínimo 4.5:1 para texto normal, 3:1 para texto grande
8. Mensagens de erro e sucesso são anunciadas por screen readers (via `role="alert"` ou `aria-live`)
9. Testes com axe-core não reportam violações CRITICAL ou SERIOUS

### Padrões de Acessibilidade (WCAG 2.2 AA)
- Usar `aria-live="polite"` para atualizações não urgentes (status de upload, resultados)
- Usar `role="alert"` ou `aria-live="assertive"` para mensagens de erro críticos (falha de conexão WhatsApp)
- Focus trapping NÃO necessário nesta story (sem modais ainda)
- `tabindex="0"` apenas para elementos não nativamente focáveis que precisam de foco
- Associar `<section>` a headings: `<section aria-labelledby="sec1-heading"><h2 id="sec1-heading">...</h2></section>`

### Dev Notes
- **Decision 1:** `aria-label` é proibido em elementos `<pre>` por WCAG/ARIA spec — removido de `#evolutionResult` e `#sendResult`; estes já têm `aria-live` como mecanismo de anúncio.
- **Decision 2:** jsdom reset `lang` attribute ao usar `documentElement.innerHTML` — restaurado manualmente no `beforeEach` do teste.
- **Decision 3:** Canvas não implementado em jsdom (warning de `axe-core` color-contrast rule) — não é falha, é limitação conhecida do jsdom; colour contrast está implementado no CSS com valores que excedem 4.5:1.
- **Decision 4:** `eslint.config.cjs` actualizado com override de globals browser para o ficheiro de teste — mínima alteração, não afecta outros testes.
- **Verificação manual de contraste (AC 7):** texto `#0f172a` sobre `#e2e8f0` (pre) → ratio ~16:1 ✓; texto `#fff` sobre `#0d6efd` (botões) → ratio ~4.6:1 ✓; texto body `#333`/`#000` sobre `#fff` → ratio ≥7:1 ✓.
- **Tab order:** Sequencial, natural — nenhum `tabindex > 0` usado.
- **`app.js` não modificado:** As live regions já recebiam conteúdo via `textContent` — nenhuma alteração necessária.
