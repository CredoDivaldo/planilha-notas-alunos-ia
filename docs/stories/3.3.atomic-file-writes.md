## Story

**As a** developer,
**I want** all JSON storage writes to use an atomic write pattern,
**so that** a process crash or interruption mid-write never leaves a corrupted or partial JSON file.

---

## Acceptance Criteria

### AC-1: Writes use temp-file-then-rename pattern
**Given:** `storage.js` is used to write any JSON data file
**When:** a write operation is performed
**Then:** the data is written to a temporary file (`.tmp` suffix in the same directory) and then renamed to the target path — never written directly to the target file

### AC-2: Temporary file is cleaned up on error
**Given:** the write to the temporary file fails (e.g., disk full)
**When:** the error occurs
**Then:** the temporary file is removed if it exists, and the original target file (if any) is left intact

### AC-3: Existing data is preserved if write fails
**Given:** a target JSON file already contains valid data
**When:** a write operation fails after creating the temp file but before the rename
**Then:** the original file is not modified and its previous content remains readable

### AC-4: All existing storage tests continue to pass
**Given:** the atomic write pattern is implemented in `storage.js`
**When:** `npm run test` is executed
**Then:** all 56 existing tests pass without modification

## Risks

| Risk | Mitigation |
|------|------------|
| `fs.rename` is atomic on same filesystem (Linux/macOS) but may not be on cross-mount writes | Temp file is always in the same `data/` directory as target — same filesystem guaranteed |
| Test isolation: tests that reset data files may be affected if `.tmp` files exist | Extend `resetDataFiles()` helper to also remove `.tmp` files |

## Dev Notes

### Current `writeJson` implementation
```javascript
async function writeJson(filePath, payload) {
  await ensureDir();
  await fs.writeFile(filePath, JSON.stringify(payload, null, 2), "utf8");
}
```

### Target implementation pattern
```javascript
async function writeJson(filePath, payload) {
  await ensureDir();
  const tmpPath = `${filePath}.tmp`;
  try {
    await fs.writeFile(tmpPath, JSON.stringify(payload, null, 2), "utf8");
    await fs.rename(tmpPath, filePath);
  } catch (error) {
    await fs.rm(tmpPath, { force: true });
    throw error;
  }
}
```

`fs.rename` is guaranteed atomic on the same filesystem. The `data/` directory is always local, so both `filePath` and `tmpPath` are on the same mount.
