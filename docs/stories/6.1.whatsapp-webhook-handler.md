## Story

**As a** student,  
**I want** to send a WhatsApp message to the platform number,  
**so that** the system receives my message and can process it to answer questions about my grades.

---

## Acceptance Criteria

### AC-1: Webhook endpoint is created and secured
**Given:** Evolution API sends a POST to `/api/v1/chatbot/webhook`  
**When:** the request includes the correct `X-Webhook-Token` header  
**Then:** the system returns `200 OK` immediately and queues the message for async processing

### AC-2: Invalid webhook token is rejected
**Given:** a request arrives at `/api/v1/chatbot/webhook`  
**When:** the `X-Webhook-Token` header is missing or incorrect  
**Then:** the system returns `401 Unauthorized` and logs the rejected attempt

### AC-3: Student is identified by phone number
**Given:** the webhook payload contains a sender phone number (`remoteJid`)  
**When:** the system processes the message  
**Then:** it queries the `students` table by normalised phone number and retrieves the matching student record

### AC-4: Unrecognised phone number is handled gracefully
**Given:** the sender phone number does not match any student  
**When:** the system processes the message  
**Then:** it logs the unknown number and does NOT call the AI service or expose any data; optionally sends a polite "number not registered" reply

### AC-5: Non-message events are ignored silently
**Given:** Evolution API sends webhook events that are not text messages (e.g., status updates, read receipts)  
**When:** the system receives such an event  
**Then:** it returns `200 OK` immediately without processing or logging an error

## Scope

### IN
- FastAPI endpoint `POST /api/v1/chatbot/webhook`.
- Webhook token validation via `CHATBOT_WEBHOOK_TOKEN` environment variable.
- Phone number normalisation (strip `@s.whatsapp.net`, handle country code).
- Student lookup by normalised phone number.
- Async dispatch to chatbot service (stub acceptable in this story).
- Logging: received message event, student identified / not found.

### OUT
- AI call or grade query (Story 6.2).
- Sending WhatsApp reply (Story 6.3).
- Rate limiting (Story 6.3).
- Multi-turn conversation state.

## Dev Notes

### Webhook payload structure (Evolution API)
```json
{
  "event": "messages.upsert",
  "instance": "instance-name",
  "data": {
    "key": {
      "remoteJid": "244912345678@s.whatsapp.net",
      "fromMe": false,
      "id": "message-id"
    },
    "message": {
      "conversation": "Qual é a minha nota de Matemática?"
    },
    "messageType": "conversation"
  }
}
```

### Phone normalisation
Strip `@s.whatsapp.net` suffix, strip leading `+` or `00`, normalise to digits only. Store in `students` table in the same normalised format.
