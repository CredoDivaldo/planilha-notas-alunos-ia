## Story Statement

**As a** professor, student, or delegate,  
**I want** a single login page with a tab switcher to choose my persona,  
**so that** I can authenticate with the correct credentials and be redirected to my specific portal without navigating to separate login URLs.

---

## Acceptance Criteria

- [x] **AC1:** A página `/login` renderiza um card centrado (max-width 440px) sobre fundo `#F8FAFC` com o título "📊 Planilha Notas"
- [x] **AC2:** Dois tabs são visíveis: "Professor" e "Estudante" (delegado usa o tab Professor com o mesmo email); o tab activo tem sublinhado azul `#0D6EFD`
- [x] **AC3:** Tab Professor mostra: campo email (type="email"), campo palavra-passe (type="password"), botão primário "Entrar" desactivado até ambos os campos preenchidos
- [x] **AC4:** Tab Estudante mostra: campo número de estudante (type="text", pattern numérico), campo palavra-passe (type="password"), botão primário "Entrar como Aluno" desactivado até ambos os campos preenchidos
- [x] **AC5:** Ao submeter, chama `POST /auth/login` com `{email_or_student_number, password, role}` e armazena o JWT no `AuthContext`
- [x] **AC6:** Após login bem-sucedido, o utilizador é redirecionado com base no `role` do JWT: `professor` → `/painel`, `estudante` → `/portal`, `delegado` → `/delegado`
- [x] **AC7:** Se as credenciais forem inválidas (HTTP 401), aparece mensagem de erro `"⚠️ Email ou palavra-passe incorrectos."` abaixo do botão, com `aria-live="assertive"`
- [x] **AC8:** Se a conta estiver suspensa (HTTP 403), aparece mensagem `"❌ Conta suspensa. Contacte o administrador."`
- [x] **AC9:** Durante o pedido, o botão muda para estado de loading com spinner e texto "A entrar…" e fica desactivado
- [x] **AC10:** Quando o backend responde com `requires_password_change: true`, o formulário de login é substituído pelo formulário de primeira troca de senha (ver AC11–AC14)
- [x] **AC11:** O formulário de primeira troca mostra: título "🔐 Primeiro Acesso", campos "Nova palavra-passe" e "Confirmar palavra-passe", indicadores inline de requisitos (mín. 8 caracteres, letra maiúscula)
- [x] **AC12:** O botão "Definir e Entrar" está desactivado até os dois campos coincidirem e os requisitos de senha estarem satisfeitos
- [x] **AC13:** Ao submeter a nova senha, chama `POST /auth/change-password` e, em caso de sucesso, redireciona para o painel correcto
- [x] **AC14:** A página é acessível WCAG AA: todos os inputs têm `<label>` associado, o focus ring é visível (outline 2px `#0D6EFD`), erros usam `aria-live`, tab order é: email/nº → senha → botão
- [x] **AC15:** Se o utilizador já estiver autenticado e navegar para `/login`, é redirecionado automaticamente para o seu painel

## Dev Notes

### Wireframe de Referência
**Ficheiro:** `outputs/wireframes/planilha-notas/screens/00-login.md`

O wireframe define:
- Card centrado 440px, fundo `#F8FAFC`
- Dois tabs: Professor (email) e Estudante (número de estudante)
- Botão "Entrar" desactivado até campos preenchidos
- Variante de primeiro acesso com indicadores de requisitos de senha

### Componentes shadcn/ui a Usar

| Componente Inventário | shadcn/ui | Uso |
|----------------------|-----------|-----|
| A01 Button | `Button` | Botão "Entrar", "Definir e Entrar" |
| A02 Input | `Input` | Campos email, password, número |
| A03 Label | `Label` | Labels de campos |
| T01 LoginPage | `Card` | Container do formulário |

### Endpoints API Relevantes

| Endpoint | Método | Uso |
|----------|--------|-----|
| `/auth/login` | POST | Autenticação inicial; retorna JWT + `requires_password_change` |
| `/auth/change-password` | POST | Troca obrigatória na primeira entrada |

### Estrutura do AuthContext (Story 7.1)

O `useAuth()` hook disponibiliza:
```ts
const { login, logout, isAuthenticated, user, role } = useAuth()
```
O `login()` recebe `{ emailOrStudentNumber, password, role }`, chama a API e guarda o token.

### Acessibilidade (WCAG AA)

- `<label htmlFor="email">Email / Utilizador</label>` com `<Input id="email" />`
- Erros de validação: `<p role="alert" aria-live="assertive">`
- Focus ring: classe Tailwind `focus-visible:outline-2 focus-visible:outline-primary`
- Indicadores de requisitos de senha: `aria-label` descritivo por cada requisito

### Notas de Estado

- **Default:** formulário vazio, botão disabled
- **Loading:** botão com `<Spinner />` + "A entrar…", desactivado
- **Erro credenciais:** `StatusMessage` variant="error" abaixo do botão
- **Conta suspensa:** `StatusMessage` variant="error" mensagem específica
- **Primeiro acesso:** substitui formulário principal; não é modal separado
