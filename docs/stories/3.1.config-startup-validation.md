## Story

**As a** system operator,
**I want** the application to validate all required environment variables at startup,
**so that** configuration errors are caught immediately with a clear message rather than causing silent runtime failures.

---

## Acceptance Criteria

### AC-1: App fails fast when required Evolution API vars are missing
**Given:** any of `EVOLUTION_BASE_URL`, `EVOLUTION_API_KEY`, or `EVOLUTION_INSTANCE` is absent or empty in the environment
**When:** `node src/server.js` is executed
**Then:** the process logs a clear error listing the missing variables and exits with code 1 before binding to any port

### AC-2: App starts normally when all required vars are present
**Given:** `EVOLUTION_BASE_URL`, `EVOLUTION_API_KEY`, and `EVOLUTION_INSTANCE` are all set to non-empty values
**When:** `node src/server.js` is executed
**Then:** the application starts successfully and the startup validation step passes silently

### AC-3: Validation runs before any route is registered
**Given:** startup validation is implemented
**When:** the validation step fails
**Then:** no HTTP routes are bound and no port is opened â€” the failure is purely at config level

### AC-4: Error message lists all missing variables at once
**Given:** multiple required vars are missing simultaneously
**When:** the process attempts to start
**Then:** a single error message lists all missing variables (not just the first one found)
