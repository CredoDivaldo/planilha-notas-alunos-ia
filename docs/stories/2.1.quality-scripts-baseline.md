## Story

**As a** developer,
**I want** a unified `npm run quality` script that aggregates test, lint, and typecheck,
**so that** I can validate code quality with a single command before committing.

## Acceptance Criteria

### AC-1: npm run test executes (ALREADY MET)
**Given:** package.json has jest configured inline
**When:** developer runs `npm run test`
**Then:** Jest executes and reports results
**Status:** ✅ Script exists: `"test": "jest --runInBand"`

### AC-2: npm run lint executes (PARTIAL)
**Given:** package.json has eslint script
**When:** developer runs `npm run lint`
**Then:** ESLint scans all .js files and reports violations
**Status:** ⚠️ Script exists BUT eslint.config.js missing (ESLint 9 requires flat config)

### AC-3: npm run typecheck executes (ALREADY MET)
**Given:** package.json has typecheck script with tsconfig.typecheck.json
**When:** developer runs `npm run typecheck`
**Then:** TypeScript checker executes with JSDoc type inference
**Status:** ✅ Script exists: `"typecheck": "tsc --noEmit -p tsconfig.typecheck.json"`

### AC-4: npm run quality aggregates all (MISSING)
**Given:** package.json has quality script
**When:** developer runs `npm run quality`
**Then:** All three scripts run in sequence (test → lint → typecheck)
**Status:** ❌ Script NOT configured - needs to be added


**Missing Components:**
1. `eslint.config.js` - ESLint 9 requires flat config format (not `.eslintrc.js`)
2. `quality` script - No aggregate command

**ESLint 9 Migration Notes:**
- ESLint 9.26.0 is installed (package.json devDependencies)
- Flat config required: `eslint.config.js` export array of config objects
- Old `.eslintrc.js` format NO longer supported
