# Story 8.2 — Port match + broadcast + WhatsApp lifecycle (P0, 8 pts)

**Branch:** `feature/8.2-match-broadcast-whatsapi`
**Status:** Draft

## Context

Story 8.1 ported the upload endpoints. Story 8.2 ports the three remaining
backend operations the React dashboard depends on (Step 3 match, Step 5
broadcast, Step 4 WhatsApp lifecycle) so the cutover in 8.5 does not break
the runtime flow.

## Acceptance Criteria

| AC | Description |
|----|-------------|
| AC1 | `POST /api/v1/grades/match` returns `{matched, unmatched, invalid_phones, items[]}` |
| AC2 | `POST /api/v1/broadcast/` returns `{portal_published, whatsapp_sent, failures, failure_list[]}` (or `{simulated: true}` for dry-run) |
| AC3 | `GET /api/v1/whatsapp/status` returns `{connected, instance_name}` |
| AC4 | `POST /api/v1/whatsapp/instance/create` and `GET /api/v1/whatsapp/instance/connect` work; dry-run returns `{simulated: true, qrcode: null}` when Evolution env vars are unset |
| AC5 | Evolution 4xx/5xx → 502 with sanitized message (Story 3.6) |
| AC6 | All endpoints cover the legacy Express paths the dashboard was calling |
