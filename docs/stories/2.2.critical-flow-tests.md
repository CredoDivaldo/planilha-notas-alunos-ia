## Story

**As a** developer,
**I want** a comprehensive test suite covering the upload → match → send critical flow plus error scenarios and edge cases,
**so that** I can refactor and extend the system with confidence, knowing regressions are caught automatically.

---

## Background

Story 2.1 established the quality baseline: `npm run test`, `npm run lint`, `npm run typecheck`, and `npm run quality` all execute successfully. Story 2.1 also validated that `tests/critical-flow.test.js` exists and passes.

Correct-course note:
- This story is no longer a near-term priority until the new Python backend, local database, and academic publication workflow foundations are defined.
- It should be revisited after the new platform backbone stories in EPIC-5 are clarified or implemented.

This story expands from 2 integration tests to a full test suite covering:
- Individual route unit tests (students, grades, match, send)
- Error scenario coverage (invalid CSV, empty data, Evolution API failures)
- Edge case handling (duplicate students, malformed rows, missing fields)
- Async error handling validation (asyncHandler wrapper — Story 2.3 prerequisite)

## Testing Strategy

**Pattern:** Follow the existing pattern in `tests/critical-flow.test.js`:
- `jest.mock("../src/services/evolution-client", ...)` for API mocking
- `supertest` for HTTP endpoint testing
- `beforeEach` + `resetDataFiles()` for isolation
- `jest.clearAllMocks()` between tests

**Coverage targets (goal, not hard requirement):**
- Routes: ≥ 80% statement coverage
- Services: ≥ 80% statement coverage
- Error paths: all branches covered

**Test file structure:**
