## Story

**As a** system administrator,
**I want** all CSV uploads to be validated against a defined schema before processing,
**so that** malformed or oversized files are rejected early with clear error messages rather than causing silent data corruption.

---

## Acceptance Criteria

### AC-1: Students CSV is validated for required columns
**Given:** a CSV file is uploaded to `POST /api/students/upload`
**When:** the file is missing all recognised identifier columns (`numero_estudante`, `numero`, `numeroAluno`, `id`) or all recognised name columns (`nome`, `aluno`)
**Then:** the server returns HTTP 400 with a descriptive error message listing the missing columns; no data is saved

### AC-2: Grades CSV is validated for required columns
**Given:** a CSV file is uploaded to `POST /api/grades/upload`
**When:** the file is missing all recognised identifier columns (`numero_estudante`, `numero`, `numeroAluno`, `id`) and all name columns (`nome`, `aluno`)
**Then:** the server returns HTTP 400 with a descriptive error message; no data is saved

### AC-3: Oversized files are rejected
**Given:** a CSV file is uploaded to any upload endpoint
**When:** the file size exceeds 2 MB
**Then:** the server returns HTTP 400 with an error indicating the size limit

### AC-4: Empty file is rejected
**Given:** an empty file (0 bytes) or a file with only a header row is uploaded
**When:** the server processes the upload
**Then:** the server returns HTTP 400 with a clear message ("Ficheiro CSV sem dados")

### AC-5: Valid files continue to work as before
**Given:** a well-formed CSV with the correct columns and at least one data row
**When:** uploaded to the respective endpoint
**Then:** the response is HTTP 200 with the processed data, unchanged from current behaviour

## Scope

### IN
- New `src/services/csv-validator.js` with `validateStudentsCsv(rows, headers)` and `validateGradesCsv(rows, headers)` functions
- Integration into `src/routes/students.js` and `src/routes/grades.js` after `parseCsv`
- File size check via multer `limits` config or explicit check on `req.file.size`
- Unit tests for the validator module
- Integration tests for the upload routes with invalid CSVs

### OUT
- Row-level data validation (e.g., checking phone number format) — that is in-scope for display logic, not upload gating
- Excel (`.xlsx`) file support
- CSV encoding detection beyond UTF-8

## Dev Notes

### Column recognition logic
Students: identifier = any of `['numero_estudante', 'numero', 'numeroAluno', 'id']`; name = any of `['nome', 'aluno']`
Grades: identifier = same as students; name = same as students

Headers are extracted from the first row of `parseCsv` output (object keys of first record).

Empty rows check runs first — when CSV has only a header row, `parseCsv` returns `[]` and no headers can be extracted, so "Ficheiro CSV sem dados" is returned before any column checks.

### File size limit
Multer configured with `limits: { fileSize: 2 * 1024 * 1024 }` in each upload route.
