# Story 8.3 — Vite proxy + unified dev script (P0, 2 pts)

**Branch:** `feature/8.3-vite-proxy-dev-script`
**Status:** Done

## Context

Story 8.1/8.2 finished the FastAPI port. Devs were still launching backend
and Vite in two separate terminals, on conflicting ports (Vite :5173, FastAPI
:8000, but legacy `vite.config.ts` proxied to :3000 — a dead Express port).
Story 8.3 unifies the dev experience: one command, one terminal, correct proxy.

## Acceptance Criteria

| AC | Description |
|----|-------------|
| AC1 | `npm run dev` (from project root) starts Vite on :5173 AND FastAPI on :8000 concurrently in a single terminal |
| AC2 | `http://localhost:5173/api/v1/health` returns 200 (Vite proxies `/api*` to :8000) |
| AC3 | Backend WhatsApp route is reachable through Vite (`http://localhost:5173/api/v1/whatsapp/status` returns 200 with JSON) |
| AC4 | When one process dies, concurrently surfaces a clear error/exit code in the other terminal output |

## Files

- EDIT `client/vite.config.ts` — replace single `/api` proxy with regex `^/(api|whatsapp|students|grades|broadcast)` → :8000
- EDIT root `package.json` — add `concurrently` to devDependencies, replace `dev` script with `concurrently -n backend,client -c blue,green "npm run dev --prefix backend" "npm run dev --prefix client"`
- NEW `backend/package.json` — minimal manifest exposing `dev` script (`cd .. && .venv/bin/python -m uvicorn backend.app.main:app --reload --port 8000`)
- NEW `docs/runbooks/dev.md` — runbook documenting the unified dev flow

## Quality Gate

- `pytest backend/tests/` → 167 passed
- `npx tsc --noEmit -p client/tsconfig.app.json` → clean
- Manual: `npm run dev` brings up both services; curls above return 200

## Notes

- AC3 wording in the original task said `/whatsapp/status`, but the actual FastAPI route is `/api/v1/whatsapp/status` (router has prefix `/api/v1`). Verified against the running server.
- `concurrently` ^9.1.0 installed via `npm install` (lock file updated).
