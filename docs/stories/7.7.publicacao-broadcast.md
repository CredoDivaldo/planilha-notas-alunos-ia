## Story Statement

**As a** professor,  
**I want** a guided 4-step wizard to review grades, select audience, choose notification channels, and confirm publication,  
**so that** grades are published to the student portal and WhatsApp notifications are sent only after explicit human confirmation, preventing accidental broadcasts.

---

- [x] **AC5:** **Passo 2 — Audiência:** RadioGroup com opções: "Todos os alunos com nota completa (N alunos)", "Apenas aprovados (N alunos)", "Apenas reprovados (N alunos)", "Selecção manual"; aviso sobre estudantes sem telefone válido (excluídos automaticamente); botões "← Revisão" e "Avançar: Canais →"
- [x] **AC6:** **Passo 3 — Canais:** lista de canais com checkboxes: WhatsApp (com badge de estado Conectado/Desconectado e contagem de destinatários com tel. válido), Email (badge "Não configurado", desactivado), "Apenas Portal (sem notificação)"; textarea editável com template de mensagem WhatsApp com placeholders `{{nome}}`, `{{disciplina}}`, `{{semestre}}`, `{{nota_final}}`, `{{resultado}}`; pré-visualização ao vivo com dados do primeiro estudante da lista; botões "← Audiência" e "Avançar: Confirmar →"
- [x] **AC7:** O botão "Avançar: Confirmar →" do Passo 3 está desactivado até pelo menos um canal ser seleccionado
- [x] **AC8:** **Passo 4 — Confirmar:** caixa de resumo com: contexto, audiência (N alunos), excluídos, canal e contagem de destinatários efectivos, visibilidade no portal; caixa de aviso âmbar com texto sobre irreversibilidade; checkbox obrigatória "Confirmo que revi as notas e autorizo a publicação."; botões "← Canais" e "🚀 Publicar e Enviar Notificações" (desactivado até checkbox marcada)
- [x] **AC9:** Ao confirmar, chama `POST /broadcast/` com payload de contexto + audiência + canal + template; durante o envio, o botão muda para "A publicar…" com spinner
- [x] **AC10:** Após conclusão, a página transita para o **Ecrã de Resultado** (não é um passo separado do stepper): mostra "✅ PUBLICAÇÃO CONCLUÍDA", 3 StatCards (Publicadas no portal, WhatsApp enviados, Falhas de envio), lista de falhas com nome + número + razão; botões "🔄 Re-enviar para falhados", "📥 Exportar relatório" e "← Voltar ao painel"
- [x] **AC11:** O botão "Re-enviar para falhados" está activo apenas se houver falhas; chama `POST /broadcast/` apenas com os estudantes que falharam
- [x] **AC12:** A página pode ser acedida a partir do botão "📤 Publicar notas" da página de Notas (Story 7.5) e do botão "⑤ Disparar" do Painel Professor (Story 7.3); o contexto activo é propagado via query param `?context={id}`
- [x] **AC13:** Se WhatsApp não estiver conectado, o canal WhatsApp no Passo 3 mostra badge vermelho "Desconectado" e checkbox desactivada; o professor é avisado com link para o Painel (Step 4) para conectar
- [x] **AC14:** A página é acessível WCAG AA: RadioGroup com `role="radiogroup"` + `aria-labelledby`; checkbox de confirmação tem label explícita; stepper tem `aria-current="step"` no passo activo

### Wireframe de Referência
**Ficheiro:** `outputs/wireframes/planilha-notas/screens/07-publicacao-broadcast.md`

O wireframe define:
- Stepper de 4 passos no topo com estados visuais (filled/check/empty)

- Passo 2: RadioGroup com contagens dinâmicas + aviso sem telefone

- Passo 4: caixa de resumo + aviso âmbar + checkbox obrigatória + botão desactivado até checkbox

### Estados do PublicationStepper (do component-inventory.md)

```ts
type StepState = 'pending' | 'active' | 'completed'
```
- `pending`: círculo vazio, texto cinza
- `active`: círculo preenchido `#0D6EFD`, texto azul, `aria-current="step"`
- `completed`: círculo com ✅ verde, texto verde, clicável para rever

### Botão "Publicar" — Regra Crítica

O botão "🚀 Publicar e Enviar Notificações" deve:
1. Ter `disabled={!confirmCheckboxChecked}` — é a única regra de bloqueio
2. Usar `aria-disabled="true"` além de `disabled` para manter focusável
3. Não ter nenhuma outra validação que o bloqueie além da checkbox

### Endpoints API Relevantes

| Endpoint | Método | Uso |
|----------|--------|-----|
| `POST /broadcast/` | POST | Publicar notas e enviar notificações |
| `GET /academic-contexts/{id}` | GET | Dados do contexto para resumo |
| `GET /grades/` | GET | Lista de notas para revisão no Passo 1 |
| `GET /whatsapp/status` | GET | Estado de ligação WhatsApp no Passo 3 (T11) — já utilizado em `QuickStatsSidebar.tsx` |

### Payload de Broadcast

```ts
// POST /broadcast/
{
  context_id: string,
  audience: 'all' | 'approved' | 'rejected' | number[], // manual = array de IDs
  channels: ('whatsapp' | 'portal_only')[],
  message_template: string // com {{placeholders}}
}
```

### Acessibilidade

- RadioGroup (Passo 2): `<fieldset>` + `<legend>Seleccionar Audiência</legend>` + `role="radiogroup"`
- Checkbox de confirmação (Passo 4): `<label htmlFor="confirm-checkbox">Confirmo que revi...</label>`
- Stepper: `<nav aria-label="Passos da publicação">` com `aria-current="step"` no passo activo
- Aviso âmbar: `role="alert"` para garantir leitura por screen readers
