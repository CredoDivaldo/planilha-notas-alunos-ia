## Story Statement

**As a** class delegate,  
**I want** a read-only view of my class's published grades and contact statuses,  
**so that** I can monitor the academic results of my peers and identify students with contact issues to report to the professor, without being able to modify any data.

---

## Acceptance Criteria

- [ ] **AC1:** A página `/delegado` é acessível apenas após login com role `delegado`; qualquer tentativa de acesso com outro role redireciona para o painel correcto desse role
- [ ] **AC2:** O `AppHeader` mostra: "📊 Planilha Notas", badge persistente âmbar "MODO DELEGADO" (cor `#B45309`, não removível), informação da turma (ex: "Turma: ING-T1  Sem. 2026/1"), nome do delegado e botão "Sair"
- [ ] **AC3:** A linha de resumo mostra 4 StatCards read-only (sem acções): Estudantes (total), Notas publicadas (contagem), Pendentes (sem nota publicada), Sem contacto (telefone ausente/inválido)
- [ ] **AC4:** A tabela de estudantes tem colunas: Nº Estudante, Nome, Disciplina, Nota, Resultado (badge colorido), Contacto (badge ✅ OK / ⚠️ Sem telefone); **sem colunas de acções, sem botões Editar ou Eliminar**
- [ ] **AC5:** A tabela mostra apenas notas com `estado = publicado`; estudantes com nota pendente mostram "—" na coluna Nota e badge "⏳ Pendente" na coluna Resultado
- [ ] **AC6:** A barra de pesquisa filtra a tabela em tempo real por nome ou número de estudante (filtragem client-side)
- [ ] **AC7:** O botão "Exportar CSV ▾" gera um CSV com a lista da turma (colunas: Nº, Nome, Disciplina, Nota, Resultado, Contacto) contendo apenas notas publicadas; notas pendentes aparecem como "—" no CSV
- [ ] **AC8:** A paginação está presente: "← Anterior  Página X de Y  Seguinte →"; 10 linhas por página
- [ ] **AC9:** O painel "Contactos com Problema" lista os estudantes com telefone ausente ou inválido com: nome, número e problema; inclui mensagem informativa "ℹ️ Comunique ao professor para corrigir estes contactos."
- [ ] **AC10:** O painel "Estado do Sistema" mostra: estado do WhatsApp (Conectado/Desconectado com badge colorido), data/hora do último broadcast, estado de publicação ("Notas publicadas: Sim/Não") e aviso "🔒 Edição de notas: bloqueada (acção exclusiva do professor)" com ícone de cadeado
- [ ] **AC11:** **Nenhum botão de edição, eliminação ou broadcast existe** em nenhuma parte da página; inspecção do DOM não deve revelar elementos ocultos com essas funções
- [ ] **AC12:** O delegado só vê dados da sua turma — o backend filtra por `turma` do JWT do delegado; o frontend não tem mecanismo de mudar de turma
- [ ] **AC13:** A página é acessível WCAG AA: badge "MODO DELEGADO" tem `aria-label="Modo de acesso: Delegado de turma, apenas leitura"`; tabela tem `<th scope="col">` e `aria-label`; StatCards têm `aria-live="polite"`

O JWT do delegado inclui o `context_id` da sua turma — o backend filtra automaticamente sem necessidade de parâmetro adicional no frontend.

### Exportação CSV Client-Side

```ts
function exportToCSV(students: Student[]) {
  const rows = [
    ['Nº', 'Nome', 'Disciplina', 'Nota', 'Resultado', 'Contacto'],
    ...students.map(s => [s.number, s.name, s.subject, s.grade ?? '—', s.result, s.contactStatus])
  ]
  const csv = rows.map(r => r.join(',')).join('\n')
  const blob = new Blob([csv], { type: 'text/csv;charset=utf-8;' })
  const url = URL.createObjectURL(blob)
  // trigger download
}
```
